# 本アプリの主な機能と設定方法の総まとめ


## 1. 本アプリの主要機能

本アプリはチャットボット形式のエージェントアプリです。
以下に記載する機能を有します。
backendはDjangoで構築し、内部サーバーを動かすことで動作します。



### 1.1 複数のプロジェクトを切り替えることができる機能

本アプリでは一つのウィンドウで複数のプロジェクトを切り替えながら作業できます。各プロジェクトはGitのワークツリーで隔離され、他のプロジェクトに影響を与えません。これは大型アップデート以前からある基本機能で、CLI/IDE拡張機能と連携して動作します。

### 1.2 Skills（スキル）とオートメーション

本アプリはエージェントスキルを有し、特定のコマンドで確認できます。
スキルとオートメーションを組み合わせることで、エラー検証やデータ分析など繰り返し作業を自動化できます。


### 1.3 サンドボックス

安全にコマンドを実行できるサンドボックスも提供され、Pythonが実行可能です。
なお、実行するプログラムに必要なライブラリはAllowリストに記載されたものだけインストールを許可します。


### 1.4 独立実行

ユーザーが別の作業をしていてもエージェントが独立して動けるようになりました。


### 1.5 その他の重要な機能

- **非コード成果物** – PDF・スプレッドシート・ドキュメント・プレゼンテーションなどの生成物を作ることができます。また、生成されたファイルは画面のサイドバーから確認できます。
- **スレッドオートメーション** – 定期的なチェックや長時間処理を行う「thread automations」を設定し、保存したコンテキスト内で自動的に作業が行えます。
- **承認とサンドボックス** – ファイル操作やコマンド実行を行う際に許可が必要になる場合、画面上で許可を求めます。ユーザーによって承認された場合にのみ実行されます。
- **Web検索ツール** – Web検索が必要な場合は検索を実行できます。検索に関わる設定は設定ファイルから設定可能です。
- **チャットとメモリ** – 特定のプロジェクトに縛られないチャットスレッドを作成でき、メモリ機能を有効にすると、将来のスレッドにコンテキストを引き継げます。
- **通知とスリープ防止** – タスク終了や承認待ちで通知を受け取れるほか、タスク実行中にPCをスリープさせない。モニターオフは許可される。


### 2.1 対話モードとセッション管理

- Webアプリが起動し、リポジトリを読み込みながらチャット形式でコード編集やコマンド実行ができます。長いチャットをコンパクト化する`/compact`などのスラッシュコマンドも利用できます。
- `/resume`で以前のセッションを再開でき、`exec resume`は非対話実行を再開します。

### 2.2 モデルと推論

使用するモデルは設定ファイルで指定可能で、OpenAI互換APIと通信を行います。`/model`コマンドで使用しているモデル名を確認できます。


### 2.3 機能フラグ管理

機能フラグがあり、`/features list`で一覧、`/features enable <feature>`または`/features disable <feature>`で有効・無効化できます。スラッシュコマンド`/experimental`も試験的機能をオンにします。


## 4. スラッシュコマンドとベストプラクティス

本アプリでは直感的に操作するためのスラッシュコマンドが多数用意されています。主なものを以下に示します。

| コマンド | 説明 |
| --- | --- |
| `/experimental` | 開発中の新機能をオン/オフにします。 |
| `/resume` | 保存した会話を再開します。 |
| `/fork` | 現在の会話を分岐し、新しいスレッドを開始します。 |
| `/compact` | 長いスレッドを要約してコンテキストを節約します。 |
| `/agent` | 複数エージェントの切り替えや管理を行います。 |
| `/theme` | TUIやアプリのハイライトテーマを変更します。 |
| `/apps` | ChatGPT Appsを開き、インストール済みアプリを利用します。 |
| `/status` | セッションの状態を表示し、利用モデルや機能フラグを確認します。 |


## 5. メモリ機能

本アプリのメモリ（Memories）は、過去のスレッドで学んだ情報やユーザーの好みを新しいスレッドに引き継ぐ仕組みです。標準では無効ですが、設定で有効にすると以下の特徴を持ちます。

- **長期的な文脈保持** – コーディング規約やお気に入りのライブラリ、プロジェクトの流儀などを覚え、次回以降のタスクに反映します。
- **生成／利用の制御** – `[memories]`セクションに`generate_memories=true`や`use_memories=true`を設定して生成と利用を制御します。`disable_on_external_context=true`で外部コンテキストがある場合はメモリを使わないようにできます。
- **スレッド単位でのオン/オフ** – `/memories`コマンドで現在のスレッドだけメモリの利用を切り替えることも可能です。

## 6. 設定と構成方法

本アプリは多くの機能をユーザーの好みに応じてカスタマイズできます。設定は主に`config.toml`ファイル、プロジェクトごとの設定ファイルから行います。

### 6.1 設定ファイルの場所と優先順位

設定ファイルは`~/.maigent/config.toml`に保存され、リポジトリ内の`.maigent/config.toml`がプロジェクト固有の上書きを行います。優先順位は「チャット内のフラグ → プロファイル値 → プロジェクト設定 → ユーザー設定 → システム設定 → 組み込みデフォルト」の順です。

### 6.2 代表的な設定オプション

- **モデルの指定** – `default_model`または`model`パラメータで使用モデルを固定します。
- **承認プロンプトとサンドボックス** – `approval_prompt`でコマンド実行時の確認方法を調整し、`sandbox_level`でローカル環境との分離度を設定します。`permission_profile`で自動承認・読み取り専用・完全アクセスなどのプロフィールを指定できます。
- **Web検索モード** – `web_search="able"/"disabled"`から選択します。
- **推論労力** – `model_reasoning_effort`で生成品質と計算コストをバランスします。
